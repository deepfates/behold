# Packaged Place to first-person life

On July 14, 2026, Behold admitted the independently built `venice-core-v1`
Place release under its vanilla-first `living` profile and ran the current
first-person resident loop inside the resulting world. This is the first proof
that the Place and inhabitant lanes meet at their intended narrow boundary.

The release remained a Place-owned immutable artifact. Behold independently
verified its checksum closure and portable world-tree digest, materialized a
profiled source, baseline, and runtime, derived world
`venice-core-9a802c78123ffd46`, and then owned the world epochs, resident body,
observations, actions, lifecycle journals, and Lync autobiography. No Place
recipe, route, compiler object, controller, mind, or identity type crossed the
boundary.

## Lived proof

`ProofResident` entered on an inspected, supported part of the Rialto bridge.
Its `behold.inhabitant.v2` observation used a fixed 45-ray visual terrain
sample, exposed one visible and supported apple as exact `entity:1`, and
offered no loaded-volume scans. The resident chose the ordinary player act
“pick up this dropped item.” Minecraft reported `playerCollect`; a fresh
`ProofWitness` connection independently observed no dropped apple.

The managed owner drained the resident, saved Minecraft, stopped the server,
closed its chained lifecycle journal, and released epoch 1. Epoch 2 then opened
the same runtime as a new managed incarnation. Minecraft restored the resident
at its changed position with one apple in inventory. Lync restored one prior
turn. The resident performed `status`, did not repeat collection, and appended
turn 2 to the same parent-linked autobiography.

All 16 `behold.packaged-place-continuity.v1` assertions pass. The observation
p95 was 0.645 ms against a 50 ms budget, with zero model calls and zero model
cost. The independent verifier checks the Place and Behold digest domains,
exact evidence-file closure, both lifecycle chains, epoch sequence, first-person
observation evidence, one-time Minecraft consequence, restart state, and the
authoritative Lync root and turn chain.

## What the failed attempt taught us

The first clean attempt used the bridge checkpoint and an older affordance
coordinate. The item fell three blocks down the bridge geometry. The newer body
navigated toward it but refused to claim collection after the exact entity left
perception during the final pickup. We retained that diagnostic run and did not
weaken the safety rule. A fresh read-only Mineflayer body inspected the nearby
surface through Minecraft, and the passing run used two same-height,
headroom-clear cobblestone columns already consistent with the Place route
evidence.

This is useful integration feedback: a geographic arrival checkpoint is not
automatically a safe action fixture, and an item's summon coordinate is not its
settled affordance. Evaluation fixtures need independently inspected support;
the normal world/package boundary does not need to grow new controller or Place
ontology to express that fact.

## Stronger regression retained

The same committed code also reran the synthetic
`behold.first-person-continuity.v1` proof. It passed all 20 assertions: bounded
body-owned travel, a tracked but visually occluded player, denial before motion,
later visual acquisition, exact pursuit of a moving player, one exact item
collection, an independent consequence witness, two managed epochs, and a
five-turn Lync chain. Its observation p95 was 0.856 ms and its independently
verified report SHA-256 is
`af5f24057d5c6bd1fc7f98947c8a9b03af23ab27f51a60ac1536aa9dde12f7a0`.

The two proofs are intentionally separate. Venice proves the real package to
life junction. The disposable flat world isolates difficult perception and
body semantics. Requiring every place to contain the synthetic occlusion maze
would make the integration less honest, not more complete.

## Claims and nonclaims

This proves that an accepted Place artifact and named runtime profile can
become a Behold-owned world epoch in which one resident perceives, acts,
experiences an independently observed consequence, and continues across a
restart. It also proves that the stronger first-person diagnostic still passes
after the integration.

It does not prove long autonomous life, model judgment, multiplayer society,
place-scale navigation, native ecology quality, or population performance.
Those remain separate demonstrations. Arrival and affordance coordinates are
evaluation inputs, not additions to the Place/Behold launch contract; ordinary
launch still needs only an accepted artifact and runtime profile.

## Reproduction

From the clean revision, use `place:epoch` to admit a checksummed release, run
`proof:owned-world` with the admitted root and independently inspected arrival
and affordance points, then use `proof:verify-owned-world` to verify or package
the result. The proof refuses a dirty source tree, a reused run root, a drifted
admission, an occupied port, a mismatched server JAR, false or extra assertions,
escaping evidence paths, malformed lifecycle chains, and malformed or
discontinuous Lync turns.
