# A real model resumed one physical project inside a packaged place

On July 14, 2026, `ProjectResident` used `openai/gpt-5.6-luna` to build one
two-block landmark across two separately managed Minecraft epochs in the
accepted Venice `living` world. This is the first proof that the real model
loop, durable project memory, and independently compiled Place artifact all
compose without a special controller.

The controller revision is `70be288a61f6df0099306ed0d20c8519651316b2`.
The full repository gate passed 289 tests before the live run.

## What happened

Behold independently admitted `venice-core-v1`, verified its release closure,
and materialized a disposable runtime. In epoch one, Luna chose to collect the
visible two-cobblestone stack, start project `spawn-landmark`, place one block
at `(1550,-34,999)`, record `(1551,-34,999)` as the unfinished adjacent step,
and yield. Minecraft confirmed the pickup and placement. A separately connected
`ProjWitnessAct` body saw the first block.

The managed owner then drained the resident, saved Minecraft, stopped the
server, and released the epoch. Epoch two reopened the same world and entity.
The resident had five prior Lync turns, one cobblestone, the first block as a
prior world consequence, and the active project. Its first action was to
restate the remaining commitment. It then placed the distinct adjacent block,
completed the project only after Minecraft confirmation, and yielded.
`ProjWitnessDone` independently saw both blocks.

The model received first-person `behold.inhabitant.v2` observations and no
loaded-world scan. All critical tool choices used provider `tool_choice: auto`;
the task named the project sequence and exact prepared cells, but no controller
forced a tool.

## The useful failure

The first fresh attempt is preserved at
`.behold-runtime/place-epoch-proofs/venice-luna-project-v1/`. Its resource was
supported but outside the resident's initial camera view. The first-person
action gate therefore withheld `collect_nearby_item`. Luna tried to place a
block, Minecraft rejected the attempt because inventory was empty, and the mind
yielded instead of claiming success. We ended the resident process deliberately;
the managed runner treated that as an unhealthy controller, saved and stopped
Minecraft, and retained the failed journal and lifecycle.

This exposed an evaluation limitation rather than a missing Mineflayer body
primitive. Behold already has the native `look_at` resident action, backed by
Mineflayer's `bot.lookAt`; this tightly scoped project allowlist did not include
it. The passing rerun put the prepared resource inside the initial field of
view. A future less-scripted project trial should admit ordinary looking and
movement and let the resident orient itself instead of preparing the camera
relationship.

## How this fits the other work

No new universal runtime was created.

- Place Compiler still owns immutable geography, recipes, profiles, packages,
  and provenance. Behold consumes only a package and profile, then owns the
  disposable world epoch.
- Minecraft owns physics, inventory, placement, mobs, time, save data, and
  consequence truth. Mineflayer is the body adapter.
- Ax remains the optional DSPy-style typed mind implementation. This proof used
  the direct OpenRouter mind through the same mind boundary.
- Published Lync 0.3.0 stores the resident's causal autobiography; Behold owns
  the meaning of its entity turns and bounded project projection.
- Cantrip's circle/gates/wards/loom/entity vocabulary, World Instrument's
  Proposal/Law/Outcome/Event path, ALMO's world hardness, and Armok's immutable
  checkpoint to writable continuation all describe the same separations from
  different media. They are design translations, not dependencies.

The Place capacity work adds one important scaling constraint: connected
protocol bodies, persistent native entities, active vanilla brains, and Behold
model inhabitants are independent costs. We must measure them separately. A
Behold-controlled population should not also pay for a full vanilla villager
brain unless that double cognition is intentional.

## What this does not prove

This was a short, task-directed integration proof. The project, exact cells,
resource, and two-life boundary were prepared. It does not prove a self-chosen
project, open-ended site selection, natural recovery from poor orientation, a
human relationship, multiple residents, long survival, city-scale cognition,
or a household. Those remain the next lived tests; the result here is that they
can now run in a real packaged place without changing the mind or world
contracts.
